Purpose

e2e/scripts/m16-live-tui-tmux-soak.sh is the M16 visual soak for
multi-agent orchestration through the real AppUI stdio path.

It launches:

- octos serve --stdio from this repo.
- octos-tui --mode protocol --stdio-command ... through the octos-tui tmux
  harness.
- A typed AppUI review/start request that asks for supervised specialist
  agents. Native specialists are resolved from
  OCTOS_REVIEW_NATIVE_SPECIALISTS_JSON, then profile
  review.native_specialists, then the built-in default template. Optional
  reviewer-cli and reviewer-mcp specialists are added when their backends are
  configured.

The backend uses the production review/start path inside octos serve --stdio.
Native model-backed specialists, a CLI-process specialist, and an MCP stdio
specialist are normalized into the same AppUI agent lifecycle. The Python
AppUI fixture backend is not used.

Command

From the octos repo:

  bash e2e/scripts/m16-live-tui-tmux-soak.sh run

Useful overrides:

  OCTOS_TUI_REPO=/Users/yuechen/home/octos-tui \
  OCTOS_M16_UX_RUN_ID=m16-local \
  bash e2e/scripts/m16-live-tui-tmux-soak.sh run

Set OCTOS_M16_BUILD=0 to skip building octos with the api feature.
Set OCTOS_M16_BUILD_TUI=1 to force rebuilding octos-tui; by default the script
uses an existing octos-tui binary and only builds it when missing.
Set OCTOS_M16_UX_KEEP_SESSION=1 to leave tmux open after the run.
