## CRITICAL — 5 findings

### C1. Unauthenticated `/metrics` endpoint

- **Category:** A01 Broken Access Control
- **File:** `octos-cli/src/api/router.rs:52`
- **Description:** The Prometheus `/metrics` endpoint is placed on the `public` router, bypassing bearer-token auth middleware. Any network-reachable client can read model names, token usage, per-tool call counts, and session cost without authentication.
- **Fix:** Move `/metrics` behind auth middleware or a separate admin token. At minimum, document firewall requirement.

### C2. PKCE verifier uses UUID (non-CSPRNG)

- **Category:** A02 Cryptographic Failures
- **File:** `octos-cli/src/auth/oauth.rs:31-44`
- **Description:** PKCE `code_verifier` is two concatenated UUIDv4 strings; OAuth `state` is one UUIDv4. RFC 7636 requires cryptographically secure randomness. The `uuid` crate's `new_v4()` uses `rand::thread_rng()` which may not be seeded from OS entropy in all environments.
- **Fix:** Use `rand::rngs::OsRng` explicitly with 32 random bytes, base64url-encoded.

### C3. `Decision::Ask` silently falls through to `Allow`

- **Category:** A03 Injection / A04 Insecure Design
- **File:** `octos-agent/src/tools/shell.rs:113-119`
- **Description:** `SafePolicy` returns `Decision::Ask` for commands like `sudo`, `rm -rf`, `git push --force`. The handler logs but executes unconditionally. The comment "coordinator can intercept" refers to non-existent code. `Ask` behaves identically to `Allow`.
- **Fix:** Treat `Ask` as `Deny` in non-interactive mode (API/gateway). Implement blocking prompt for CLI.

### C4. Sandbox disabled by default

- **Category:** A04 Insecure Design
- **File:** `octos-agent/src/sandbox.rs:59-67`
- **Description:** `SandboxConfig::default()` sets `enabled: false`. Combined with C3, all default-config users execute LLM-directed shell commands unsandboxed with no effective approval gate.
- **Fix:** Default to `enabled: true`, `mode: SandboxMode::Auto`. Warn prominently when no backend found.

### C5. DNS rebinding window in SSRF check

- **Category:** A10 SSRF
- **File:** `octos-agent/src/tools/web_fetch.rs:100-127`
- **Description:** SSRF validates DNS then `reqwest` re-resolves independently. DNS rebinding between the two lookups bypasses private IP check. Same race exists in `browser.rs`.
- **Fix:** Use `reqwest`'s custom DNS resolver with IP filtering at the socket-connect level, or connect directly to pre-resolved IP.

## HIGH — 10 findings

### H1. Mutable tool state in concurrent sessions

- **Category:** Logical race condition
- **File:** `octos-agent/src/tools/message.rs:14-16`, `spawn.rs:24`
- **Description:** `MessageTool` and `SpawnTool` carry per-request context (`default_channel`, `default_chat_id`) behind `std::sync::Mutex`, shared across concurrent gateway sessions via `Arc<ToolRegistry>`. Two sessions can race on `set_context()`, causing replies to go to the wrong channel. **Most dangerous active logical bug.**
- **Fix:** Pass routing context through tool arguments or a per-request `ToolContext`, not shared mutable state.

### H2. MCP HTTP transport has no SSRF protection

- **Category:** A10 SSRF
- **File:** `octos-agent/src/mcp.rs:413-430`
- **Description:** HTTP MCP server URLs connect via bare `reqwest::Client::new()` with no SSRF check. Malicious config can reach cloud metadata endpoints (AWS IMDS, etc.).
- **Fix:** Apply `ssrf::is_private_host` + DNS check to all MCP HTTP URLs.

### H3. MCP tool name collision allows registry poisoning

- **Category:** A01 Broken Access Control
- **File:** `octos-agent/src/mcp.rs:330-348`
- **Description:** `HashMap::insert` silently overwrites existing tools. A malicious MCP server advertising `shell` or `write_file` replaces built-in safe implementations.
- **Fix:** Prefix MCP tool names with server name. Refuse to register names conflicting with built-ins.

### H4. `~username/` tilde expansion unsanitized

- **Category:** A01 Broken Access Control
- **File:** `octos-agent/src/hooks.rs:354-366`
- **Description:** Username interpolated without validation. `~../../../etc/` expands to `/Users/../../../etc/`.
- **Fix:** Validate username contains only `[a-zA-Z0-9_-]`.

### H5. `web_fetch` buffers full response before truncation

- **Category:** A09 DoS
- **File:** `octos-agent/src/tools/web_fetch.rs:155-170`
- **Description:** `response.text().await` buffers entire body. `max_chars` is user-controlled with no upper bound. GB-sized responses cause OOM before truncation runs.
- **Fix:** Clamp `max_chars` to 500,000 max. Stream response body with byte limit.

### H6. Multiple System messages dropped for Anthropic

- **Category:** Correctness
- **File:** `octos-llm/src/anthropic.rs:76-81`
- **Description:** `.find()` picks only the first System message. Memory context injected as additional System messages is silently lost for Anthropic (works for OpenAI/Gemini).
- **Fix:** Concatenate all System messages before calling Anthropic API.

### H7. `DefaultHasher` for session filenames is unstable

- **Category:** A08 Data Integrity
- **File:** `octos-bus/src/session.rs:183-188`
- **Description:** `std::collections::hash_map::DefaultHasher` provides no stability guarantees across Rust versions. Sessions with long keys become inaccessible after compiler upgrades.
- **Fix:** Use `sha2::Sha256` (already a workspace dependency) truncated to 8 bytes.

### H8. `build_result` always reports `success: true`

- **Category:** Correctness
- **File:** `octos-agent/src/agent.rs:938-954`
- **Description:** `build_result` hard-codes `success: true` even on `MaxTokens` stop. Callers get incorrect status for incomplete tasks.
- **Fix:** Pass stop reason into `build_result`.

### H9. Provider instantiation duplicated

- **Category:** Maintainability
- **File:** `octos-cli/src/commands/gateway.rs:121-247`, `chat.rs`
- **Description:** Provider name -> `Arc<dyn LlmProvider>` mapping duplicated. New providers require edits in two places.
- **Fix:** Extract `create_provider()` into shared module.

### H10. `TokenUsage` defined in two crates

- **Category:** Maintainability
- **File:** `octos-core/src/task.rs:141`, `octos-llm/src/types.rs:34`
- **Description:** Structurally identical types with manual field copy. Will silently diverge if fields added.
- **Fix:** Define once in `octos-core`, re-export from `octos-llm`.

## Methodology

Three parallel review agents were used:

1. **Security reviewer** — OWASP-focused vulnerability analysis across all tool, sandbox, auth, and network code paths
2. **Code quality reviewer** — Error handling, API design, testing gaps, Rust idioms, performance patterns
3. **Architecture reviewer** — Crate boundaries, trait design, async patterns, state management, extensibility

Additionally, automated checks were run: `cargo test`, `cargo clippy`, `cargo fmt`, `cargo audit`, plus manual grep analysis for `unsafe`, `unwrap`, `panic!`, `todo!`, `FIXME`, and file size metrics.
