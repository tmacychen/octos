Features

Everything you need in an AI agent

A complete agent framework with tool use, memory, multi-provider failover, and messaging channels—all in a single binary.

🧠

Multi-Provider LLM

Supports 12+ LLM providers with automatic failover. OpenAI, Anthropic, Gemini, Moonshot, DeepSeek, and more. Switch models without changing code.

🛠️

Tool System

Shell, file I/O, web search, browser automation, code analysis. Sandboxed execution with configurable policies. Extend with custom MCP servers.

💬

Messaging Channels

Connect via Telegram, Feishu/Lark, WhatsApp, Twilio SMS, Discord, Slack, WeChat, email, or the built-in web chat. All channels, one agent.

💾

Persistent Memory

Hybrid BM25 + vector search over conversation history. Episode store for task summaries. Your agent remembers across sessions.

🌐

Admin Dashboard

Web-based control panel for managing profiles, channels, API keys, and gateway processes. Monitor status, view logs, restart services.

🔒

Private & Secure

Runs on your hardware. Your data never leaves your machine. Sandboxed tool execution, symlink-safe file I/O, SSRF protection built in.

Cloud Onboarding

Up and running in 3 steps

An admin provisions your account, bootstraps your Mac Mini remotely, and you get a fully functional AI dashboard accessible from anywhere.

Create Account

Admin creates a cloud tenant for you. This assigns a subdomain, SSH tunnel port, and generates your dashboard auth token.

Bootstrap Mac Mini

A single script installs octos, frpc tunnel client, configures services, and starts everything. Works on fresh macOS or Linux machines.

Go Live

Your dashboard is immediately accessible at your subdomain. Add LLM API keys, connect messaging channels, and start chatting.

# Step 1: Admin creates your cloud account
octos admin create-tenant --name alice

  Tenant created:
    Subdomain:  alice.octos-cloud.org
    Auth token: a1b2c3d4e5f6...
    SSH port:   6001

# Step 2: Bootstrap the Mac Mini (one command)
./scripts/frp/bootstrap-tenant.sh alice cloud@10.0.1.50 --password ****

  ==> Building octos (release)...
  ==> Uploading binaries...
  ==> Installing frpc tunnel...
  ==> Starting services...
  ==> Tunnel: CONNECTED

# Step 3: Done! Dashboard is live
open https://alice.octos-cloud.org
Architecture

How it connects

Your Mac Mini connects outbound to our relay server via frp tunnel. No port forwarding or public IP needed. Traffic is routed by subdomain. 

  Internet
     |
     v
  Caddy  (octos-cloud.org, SSL termination)
     |
     v
  frps   (tunnel relay, subdomain routing)
     |         |         |
     v         v         v
  alice     bob       carol      <-- frpc tunnels (outbound)
     |         |         |
     v         v         v
  Mini:8080 Mini:8080 Mini:8080   <-- octos serve (dashboard + API)
     |         |         |
     v         v         v
  Telegram  Feishu   WhatsApp     <-- messaging channels
⚡

Rust-Native Performance

Single binary, ~30MB. Starts in milliseconds. Handles hundreds of concurrent sessions with minimal memory. No runtime dependencies.

🔄

Auto-Failover

Configure primary + fallback LLM providers. 401/403 triggers failover. 429/5xx triggers retry then failover. Your agent stays online.

📋

Multi-Profile

Run multiple agent profiles on one machine. Each with its own LLM provider, channels, system prompt, and data directory. Fully isolated.
