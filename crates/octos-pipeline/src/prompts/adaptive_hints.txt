- Search strategy: deep_search supports multiple engines via search_engine parameter:
  * 'perplexity' — AI-powered, best quality, returns synthesized answers with citations (paid per call)
  * 'tavily' — AI-optimized, fast, good factual queries (1000 free/month, then paid)
  * 'brave' — free, fast, good general purpose
  * 'duckduckgo' — free but slow/unreliable
  * omit search_engine → races top 2 engines in parallel, picks richest result (good default)
  * 'all' → fires ALL engines in parallel (max coverage, use sparingly on 1-2 key angles)
  Most workers should omit search_engine (default top-2 racing is fast and reliable).
- Search angles: 3-4 for simple topics, 5-8 for complex/multi-faceted topics

- Synthesize: MUST set max_output_tokens high enough for the expected report length (default 4096 truncates long reports)
- Match report language to query language
- Model selection: use cheap/fast models for search nodes, high max_output_tokens models for synthesize nodes
- Timeouts: synthesize=900s, search=600s, analyze=300s
- Analyze nodes should use tools="" (no tools) — pure text analysis
